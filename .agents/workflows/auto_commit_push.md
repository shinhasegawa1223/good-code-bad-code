# 🤖 コミット・プッシュ規約 (都度確認)

ユーザーからの作業依頼（プロンプト）によってファイルへの追加・修正・削除などの変更が加わった場合、**勝手にコミットやプッシュを行わず、必ずユーザーに「git commitとpushを実行して良いか」を確認すること**。

1. 変更内容をファイルに書き込む
2. ユーザーへ作業完了を報告し、「コミットおよびプッシュを実施してもよろしいでしょうか？」と確認する
3. ユーザーから許可が出た場合のみ、以下のコマンドで保存・反映する
   - `git add .` (または適切な変更ファイル)
   - `git commit -m "変更内容を簡潔に表すコミットメッセージ"`
   - `git push`

※ `// turbo-all` 等による無断での自動実行は行わないこと。
