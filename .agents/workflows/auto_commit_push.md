# 🤖 自動コミット・プッシュ規約

ユーザーからの作業依頼（プロンプト）によってファイルへの追加・修正・削除などの変更が加わった場合、作業の完了報告を行う前に**必ず以下のステップを実行し、変更内容をリモートリポジトリに保存すること**。

1. `git status` 変更されたファイルを確認する
2. `git add .` (または適切な変更ファイル) をステージングする
3. `git commit -m "変更内容を簡潔に表すコミットメッセージ"` を実行する
4. `git push` を実行し、リモートのブランチに変更を反映する
5. ユーザーへの完了報告メッセージには、「`git push` まで確認・完了したこと」を必ず明記する
