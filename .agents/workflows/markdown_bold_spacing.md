# 📝 Markdown太字（ボールド）スペース規約

Markdownファイルに追記・修正を行う際、**太字（ボールド）**の装飾を施す場合は、前後の文字との間に必ず半角スペースを挿入すること。

## 理由
日本語の文章全体の中で一部を太字にする際、文字の直後に `**` を記述すると、VS Codeなどの一部の環境のMarkdownプレビューで正常に太字としてレンダリングされず、そのままアスタリスクが表示されてしまう問題を防ぐため。

## ルール
- ❌ 悪い例: `変数が**「何を表すのか」**を明確にする`
- ⭕️ 良い例: `変数が **「何を表すのか」** を明確にする`

Markdownを出力する際は、装飾が確実に反映されるよう、この前後の半角スペースの規則を遵守すること。
